Privacy Policy

Last Updated: December 28, 2025

Bot Version: 2.0.0

This Privacy Policy explains how the Buck Bot ("the Bot") handles data within your Discord server.

1. Data We Collect

We believe in minimal data collection. The Bot only processes the following information to function:

  • User IDs: To identify users for moderation actions (warnings, bans, kicks, timeouts), to track authorized users for restricted commands (e.g., the download command), and to associate actions with users.
  • Guild/Server IDs: To store server-specific settings such as log channels, and to manage per-server music queues.
  • Channel IDs: To know where to play music, send logs, and execute commands.
  • Command Content: To process your requests (e.g., search queries for music, moderation reasons).
  • Message Metadata: For the message logging feature (if enabled by server admins), we temporarily process message content to generate logs. We do not store message content on external databases.
  • Voice State: To manage music playback and track which users are in voice channels.

2. Data We Store

The following data is persistently stored in a local database file:

  • Server Settings: Log channel IDs and other configuration options set by administrators.
  • Warning Records: When a user is warned, we store their User ID, the reason, the moderator's ID, and a timestamp. This data is used for the warning system and auto-timeout features.
  • Authorized Users: A list of User IDs authorized to use restricted commands like /download.

3. How We Use Data

Your data is used solely for the following purposes:

  • To facilitate music streaming, searching, and playback from the connected Navidrome server.
  • To provide moderation tools including bans, kicks, timeouts, warnings, purge, and slowmode.
  • To enforce automatic actions (e.g., auto-timeout after reaching warning thresholds).
  • To update the Bot's status with the currently playing track.
  • To provide server activity logs in the designated channel set by your administrators.
  • To restrict access to certain commands to authorized users only.

4. Data Retention

Most data processed by the Bot is ephemeral and stored in memory only while the Bot is running (e.g., music queues, current playback state). Persistent data (settings, warnings, authorized users) is stored until manually deleted by a server administrator or the Bot owner.

5. Data Security

All persistent data is stored locally on the Bot's host server. We do not sell, share, or transfer your data to third parties. Access to stored data is limited to the Bot owner for maintenance purposes.

6. Third-Party Services

The Bot interacts with the following third-party services:

  • Discord: To deliver our services to your server.
  • Navidrome: To fetch music, albums, artists, playlists, and metadata from your private media server.
  • LrcLib: To fetch song lyrics.
  • Spotify API: To fetch metadata for music downloads via SpotiFLAC.

7. Your Rights

You have the following rights regarding your data:

  • Access: Server administrators can view warnings using /warn list.
  • Deletion: Administrators can clear warnings using /warn clear. To request complete deletion of all stored data, contact the Bot owner.
  • Opt-out: Administrators can disable logging features or remove the Bot from the server entirely to stop all data processing.

8. Children's Privacy

The Bot is not intended for users under the age of 13 (or the minimum age of digital consent in your jurisdiction). We do not knowingly collect data from children.

9. Changes to This Policy

We may update this policy occasionally to reflect new features or legal requirements. Continued use of the Bot after changes constitutes acceptance of the new policy. Significant changes will be announced where possible.
